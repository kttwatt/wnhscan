# Department Page KISS Redesign

## Goal

ปรับหน้า `/department` ให้เรียบง่าย โฟกัส 2 งานหลัก:
1. เพิ่ม/ลบรายการวัสดุจากฐานข้อมูลกลาง
2. ค้นหาและพิมพ์สมุดรายการวัสดุ

## Approach

**แนวทาง A — หน้าเดียว ไม่มีแท็บ**

- ลบ `PageTabs` (จัดการรายการวัสดุ / ผู้ใช้ในหน่วยงาน)
- ผู้ใช้ในแผนกย้ายเป็นลิงก์ใน toolbar → เปิด modal อ่านอย่างเดียว

## Layout

```
PageHeader (ชื่อแผนก + DepartmentSwitcher)
└── card-whitespace
    ├── Toolbar แถวเดียว
    │   ├── [เพิ่มจากฐานข้อมูล] [สมุดรายการวัสดุ]
    │   ├── ค้นหา
    │   ├── หมวดย่อย (dropdown เดียว)
    │   └── ผู้ใช้ในแผนก (N)
    ├── แสดง N รายการ
    └── กลุ่มหมวดขยายทั้งหมด → ตาราง
```

## UX changes

| เดิม | ใหม่ |
|------|------|
| 2 แท็บ | หน้าเดียว |
| กล่อง gradient สำหรับค้นหา | toolbar แถวเดียวกับปุ่ม |
| กรอง หมวดใหญ่ + หมวดย่อย | หมวดย่อยอย่างเดียว |
| กลุ่มยุบค่าเริ่มต้น | ขยายทุกกลุ่มเมื่อโหลด/กรอง |
| ปุ่มเพิ่มซ้ำใน empty state | ข้อความอย่างเดียว (ปุ่มอยู่ toolbar) |
| แท็บผู้ใช้ | modal จากลิงก์ใน toolbar |

## Modals (คงเดิม)

- `CatalogAssignModal` — เพิ่มจากฐานข้อมูลกลาง
- `CatalogPrintPreview` — สมุดรายการวัสดุ
- `DepartmentUsersModal` — รายชื่อผู้ใช้ในแผนก (อ่านอย่างเดียว)

## Out of scope

- URL sync `?tab=`
- แก้ data model / RLS
- จัดการผู้ใช้จากหน้านี้ (ไปที่ `/admin/users`)

## Success criteria

1. เปิดหน้าแล้วเห็นรายการวัสดุทันที
2. เพิ่ม/ลบ/ค้นหา/พิมพ์สมุด ทำได้ในหน้าเดียว
3. UI ไม่เกิน 2 ชั้น visual (header + card)
4. ผู้ใช้ในแผนกเข้าถึงได้ผ่าน modal
